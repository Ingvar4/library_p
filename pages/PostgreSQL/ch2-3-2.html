Фильтрация с LIKE, IN, BETWEEN в PostgreSQL

Фильтрация данных — это процесс отбора нужных записей из таблиц с использованием условий. В PostgreSQL для этого используются различные операторы. Сегодня мы рассмотрим такие операторы фильтрации как LIKE, IN и BETWEEN.

1. Оператор LIKE

Оператор LIKE используется для поиска строковых данных, которые соответствуют заданному шаблону. Он позволяет искать строки с частичным совпадением.

Символы шаблонов:

  • % — заменяет любое количество символов (включая ноль).
  • _ — заменяет ровно один символ.

Примеры:

1. Найти все записи, где имя пользователя начинается с "john":

SELECT * FROM users WHERE username LIKE 'john%';

2. Найти все записи, где email оканчивается на ".com":

SELECT * FROM users WHERE email LIKE '%.com';

3. Найти все записи, где имя состоит из пяти символов и начинается с "A":

SELECT * FROM users WHERE username LIKE 'A____';

2. Оператор IN

Оператор IN используется для фильтрации строк, где значение поля совпадает с одним из элементов списка. Это упрощает запись, когда нужно проверить соответствие сразу нескольким значениям.

Примеры:

1. Найти всех пользователей, которые живут в Москве, Санкт-Петербурге или Казани:

SELECT * FROM users WHERE city IN ('Москва', 'Санкт-Петербург', 'Казань');

2. Найти все записи с определёнными идентификаторами (id):

SELECT * FROM users WHERE id IN (1, 5, 10, 15);

Оператор IN особенно удобен, когда нужно проверить на совпадение сразу с несколькими значениями, вместо использования нескольких условий OR.

3. Оператор BETWEEN

Оператор BETWEEN используется для фильтрации строк, где значение поля попадает в определённый диапазон. Он работает как для числовых данных, так и для дат.

Примеры:

1. Найти всех пользователей в возрасте от 18 до 30 лет включительно:

SELECT * FROM users WHERE age BETWEEN 18 AND 30;

2. Найти все заказы, сделанные в период с 1 января 2024 года по 31 марта 2024 года:

SELECT * FROM orders WHERE order_date BETWEEN '2024-01-01' AND '2024-03-31';

Здесь важно помнить, что BETWEEN включает граничные значения, то есть запись с возрастом 18 и 30 также будет включена в результат.

Заключение

Операторы LIKE, IN и BETWEEN позволяют создавать мощные условия фильтрации данных в PostgreSQL. Эти операторы помогают находить записи на основе шаблонов, списков значений и диапазонов, что делает запросы более гибкими и точными. Использование данных операторов является важной частью работы с реляционными базами данных и управления данными.

Описание основных операторов:

  1. LIKE — для поиска по шаблону (использует символы подстановки % и _).
  2. IN — для поиска значений, которые совпадают с одним из указанных в списке.
  3. BETWEEN — для поиска значений, находящихся в определённом диапазоне, включая граничные значения.
