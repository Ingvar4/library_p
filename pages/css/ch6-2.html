Использование медиа-запросов (@media) для различных устройств.

Медиа-запросы — это мощный инструмент в CSS, который позволяет изменять стили в зависимости от характеристик устройства, например, ширины или высоты экрана, ориентации, разрешения и других особенностей. Они являются основой адаптивной верстки и обеспечивают правильное отображение сайтов на различных устройствах.

Основы медиа-запросов

Медиа-запросы позволяют определить условия, при которых будут применяться определенные стили. Синтаксис медиа-запроса следующий:

  
    /*CSS*/

    @media (условие) {
      /* Стили */
    }

Пример простого медиа-запроса

  
    /*CSS*/

    @media (max-width: 600px) {
      body {
        background-color: lightblue;
      }
    }

Этот пример применяет светло-голубой цвет фона для body, если ширина экрана меньше или равна 600 пикселям.

Основные параметры медиа-запросов

  1. Ширина и высота устройства
    • width и height: задают ширину и высоту окна просмотра.
    • min-width и max-width: используются для создания брейкпоинтов.
    • Примеры:
      •   
    /*CSS*/

    @media (min-width: 768px) {
      /* Стили для планшетов и выше */
    }
    
    @media (max-width: 480px) {
      /* Стили для смартфонов */
    }
  2. Ориентация устройства
    • orientation: определяет ориентацию устройства, может принимать значения portrait (портретная) и landscape (ландшафтная).
    3.   
    /*CSS*/

    @media (orientation: landscape) {
      /* Стили для ландшафтной ориентации */
    }
  3. Разрешение экрана
    • resolution: используется для работы с плотностью пикселей, например, для устройств с высокой плотностью пикселей (ретина-дисплеи).
    4.   
    /*CSS*/

    @media (min-resolution: 192dpi) {
      /* Стили для устройств с высоким разрешением */
    }

Часто используемые брейкпоинты

В адаптивной верстке обычно выделяют несколько стандартных брейкпоинтов:

  1. min-width: 1200px: большие экраны (десктопы)
  2. min-width: 992px и max-width: 1199px: средние экраны (ноутбуки, небольшие десктопы)
  3. min-width: 768px и max-width: 991px: планшеты (портретная ориентация)
  4. max-width: 767px: смартфоны (портретная ориентация)

Пример использования:

  
    /*CSS*/

    /* Стили по умолчанию для десктопов */
    body {
      font-size: 18px;
    }

    @media (max-width: 991px) {
      body {
        font-size: 16px; /* Стили для планшетов */
      }
    }

    @media (max-width: 767px) {
      body {
        font-size: 14px; /* Стили для смартфонов */
      }
    }

Комбинирование условий медиа-запросов

  1. Использование логических операторов
    • and: объединяет несколько условий.
      •   
    /*CSS*/

    @media (min-width: 600px) and (max-width: 1200px) {
      /* Стили для устройств с шириной экрана от 600 до 1200 пикселей */
    }
    • not: исключает определенные условия.
      •   
    /*CSS*/

    @media not (orientation: landscape) {
      /* Стили для портретной ориентации */
    }
    • only: применяется для поддержки старых браузеров, которые не понимают медиа-запросы.
      •   
    /*CSS*/

    @media only screen and (max-width: 600px) {
      /* Стили для экранов шириной до 600 пикселей */
    }
  2. Множественные условия через запятую (или "или")
    • Применение стилей, если выполняется хотя бы одно из условий.
      •   
    /*CSS*/

    @media (max-width: 600px), (orientation: landscape) {
      /* Стили для экранов шириной до 600 пикселей или для ландшафтной ориентации */
    }

Подходы к адаптивной верстке с медиа-запросами

  1. Mobile-First - Мобильный в первую очередь
    • Стили для мобильных устройств пишутся по умолчанию, а стили для больших экранов добавляются с помощью медиа-запросов.
    2.   
    /*CSS*/

    /* Стили по умолчанию (для мобильных устройств) */
    .container {
      font-size: 14px;
    }

    @media (min-width: 768px) {
      .container {
        font-size: 16px; /* Стили для планшетов */
      }
    }

    @media (min-width: 1200px) {
      .container {
        font-size: 18px; /* Стили для десктопов */
      }
    }
  2. Desktop-First - Десктоп в первую очередь
    • Стили для больших экранов пишутся по умолчанию, а стили для мобильных устройств добавляются с медиа-запросами.
    3.   
    /*CSS*/

    /* Стили по умолчанию (для десктопов) */
    .container {
      font-size: 18px;
    }

    @media (max-width: 1200px) {
      .container {
        font-size: 16px; /* Стили для планшетов */
      }
    }

    @media (max-width: 768px) {
      .container {
        font-size: 14px; /* Стили для мобильных устройств */
      }
    }

Особенности использования медиа-запросов

  1. Поддержка медиа-запросов в браузерах
    • Медиа-запросы поддерживаются всеми современными браузерами. Однако важно проверять корректность отображения стилей в старых версиях браузеров, если требуется поддержка.
  2. Организация медиа-запросов
    • Вы можете писать медиа-запросы в конце файла стилей или внутри определенных селекторов. Рекомендуется использовать один из подходов на всем проекте для удобства.
    • Пример внутри селекторов:
      •   
    /*CSS*/

    .header {
      font-size: 24px;
    }
    
    @media (max-width: 600px) {
      .header {
        font-size: 18px;
      }
    }
    • Пример в конце файла:
      •   
    /*CSS*/

    .header {
      font-size: 24px;
    }
    
    @media (max-width: 600px) {
      .header {
        font-size: 18px;
      }
    }
  3. Отладка медиа-запросов
    • Используйте инструменты разработчика в браузерах, чтобы быстро переключаться между различными размерами экрана и проверять стили.

Советы по оптимизации медиа-запросов

  1. Используйте относительные единицы: вместо пикселей часто рекомендуется использовать em или rem, чтобы стили были более гибкими.
  2. Ограничьте количество брейкпоинтов: определите ключевые точки, чтобы избежать излишнего количества медиа-запросов.
  3. Тестируйте на реальных устройствах: эмуляторы полезны, но реальные устройства могут вести себя иначе.

Эта информация охватывает все важные аспекты медиа-запросов, обеспечивая правильное понимание и использование для создания адаптивных веб-страниц.
