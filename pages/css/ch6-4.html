Работа с резиновыми единицами: em, rem, vw, vh, %.

Веб-разработка часто требует гибкости в макете, чтобы сайт адаптировался к различным размерам экранов и устройствам. Резиновые единицы, такие как em, rem, vw, vh, и %, обеспечивают масштабируемость элементов и делают макеты более отзывчивыми и адаптивными. Рассмотрим подробно каждую из этих единиц.

1. Единицы em

  • Описание: em — это относительная единица измерения, основанная на размере шрифта родительского элемента.
  • Применение: Используется для масштабирования шрифта, отступов, полей и других размеров.
  • Пример:
    •   
    /*CSS*/

    body {
      font-size: 16px;
    }
    p {
      font-size: 1.5em; /* 1.5 × 16px = 24px */
    }
  • Особенности: Размер в em зависит от текущего контекста. Например, если родительский элемент имеет размер шрифта 20px, то 1em будет равен 20px, что делает em каскадной единицей.

Преимущества:

  • Гибкость: позволяет легко масштабировать элементы на основе размера шрифта.
  • Удобно для создания резиновых макетов, где элементы изменяются в зависимости от родителя.

Недостатки:

  • Сложность вычислений: из-за каскадного влияния может быть трудно предсказать итоговый размер.

2. Единицы rem

  • Описание: rem (root em) — это единица измерения, основанная на размере шрифта корневого элемента (html).
  • Пример:
    •   
    /*CSS*/

    html {
      font-size: 16px;
    }
    h1 {
      font-size: 2rem; /* 2 × 16px = 32px */
    }
  • Особенности: В отличие от em, rem не каскадирует и всегда привязана к корневому размеру шрифта. Это делает расчёты более предсказуемыми и удобными.

Преимущества:

  • Простота в управлении: размеры элементов остаются постоянными независимо от вложенности.
  • Легче масштабировать весь интерфейс, изменяя только размер шрифта корневого элемента.

Недостатки:

  • Меньше гибкости по сравнению с em в специфических ситуациях, где нужна каскадность.

3. Единицы %

  • Описание: % — это относительная единица, которая зависит от родительского элемента.
  • Применение: Используется для ширины, высоты, отступов и полей.
  • Пример:
    •   
    /*CSS*/

    .container {
      width: 80%; /* Ширина контейнера составляет 80% от ширины родительского элемента */
    }
  • Особенности: В процентах размеры могут быть относительно контейнера или родительского элемента, что позволяет гибко управлять макетом.

Преимущества:

  • Отлично подходит для создания адаптивных макетов.
  • Гибкость в использовании для определения ширины и высоты.

Недостатки:

  • Размеры могут быть непредсказуемыми в зависимости от контекста.

4. Единицы vw и vh

  • Описание:
    • vw (viewport width) — 1 единица равна 1% ширины окна просмотра.
    • vh (viewport height) — 1 единица равна 1% высоты окна просмотра.
  • Пример:
    •   
    /*CSS*/

    .full-screen {
      width: 100vw; /* 100% ширины окна */
      height: 100vh; /* 100% высоты окна */
    }
  • Особенности: Эти единицы удобны для создания макетов, которые занимают всё окно просмотра или масштабируются в зависимости от размера экрана.

Преимущества:

  • Идеально подходит для адаптивных дизайнов, особенно для полноэкранных элементов.
  • Простота управления размерами в зависимости от окна просмотра.

Недостатки:

  • Может вызывать проблемы с прокруткой при неправильном использовании, особенно на мобильных устройствах с динамическими панелями (например, адресной строкой).

Сравнение и лучшие практики

  1. Когда использовать em:
    • Для шрифтов и компонентов, которые должны наследовать размер от родительских элементов.
    • Для создания масштабируемых компонентов внутри других элементов.
  2. Когда использовать rem:
    • Для шрифтов, отступов, полей, чтобы добиться большей предсказуемости.
    • При необходимости быстро изменять весь макет, изменяя только корневой размер шрифта.
  3. Когда использовать %:
    • Для ширины и высоты элементов, чтобы создать отзывчивый и гибкий макет.
  4. Когда использовать vw и vh:
    • Для создания элементов, которые должны быть адаптивными в зависимости от окна просмотра.
    • Например, для полноэкранных фонов, секций, которые заполняют всё пространство экрана.

Заключение

Использование резиновых единиц — это ключ к созданию адаптивного и отзывчивого дизайна. Понимание, как и когда применять каждую единицу, помогает строить современные интерфейсы, которые адаптируются к различным размерам экранов.
