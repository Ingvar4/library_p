Свойства display: block, inline-block, none.

1. Использование display: inline-block

Свойство display: inline-block сочетает преимущества блочных и строчных элементов:

  • Элемент остается в одной строке с другими строчными элементами, но при этом можно задавать ему размеры: ширину (width) и высоту (height), а также отступы и поля.
  • Часто используется для создания элементов интерфейса, таких как кнопки, меню навигации и карточки с информацией.

Пример использования inline-block:

  
    <!-- HTML -->

    <div style="display: inline-block; width: 350px; height: 100px; background-color: lightcoral;">
      Я — элемент с `display: inline-block`
    </div>

    <div style="display: inline-block; width: 100px; height: 100px; background-color: lightblue;">
      Я тоже!
    </div>

Эти два <div> будут располагаться бок о бок.

Результат:

Я — элемент с `display: inline-block`
Я тоже!

2. Гибкость верстки с display

Переопределение типа элемента

  • Порой полезно изменять поведение элементов с помощью CSS. Например, превращать блочные элементы в строчные и наоборот.
  • Это удобно для улучшения структуры и стилизации контента без изменения HTML-разметки.

Пример: Превращение абзаца в строчный элемент

  
    /*CSS*/

    p {
      display: inline;
    }

Теперь <p> не будет начинать новую строку.

3. Работа с отступами у строчных элементов

Строчные элементы не учитывают вертикальные отступы (сверху и снизу) так же, как блочные. Однако горизонтальные отступы (слева и справа) работают корректно.

Пример проблемы с отступами:

  
    <!-- HTML -->

    <a href="#" style="margin-top: 120px;">Ссылка с вертикальным отступом</a>

Результат:

Ссылка с вертикальным отступом

В этом случае вертикальный отступ может не вести себя так, как ожидается, из-за особенностей поведения строчных элементов. Чтобы решить эту проблему, можно использовать display: inline-block.

4. Белые промежутки между строчными элементами

Когда строчные или inline-block элементы размещаются рядом в HTML, между ними могут появляться белые пробелы. Это связано с тем, что разметка в HTML учитывает пробелы, переносы строк и табуляции.

Пример проблемы:

  
    <!-- HTML -->

    <div style="display: inline-block;">Элемент 1</div>
    <div style="display: inline-block;">Элемент 2</div>

Результат:

Элемент 1
Элемент 2

В этом случае между элементами будет пробел. Чтобы избавиться от него:

  • Удалите пробелы в HTML-разметке.
  • Используйте отрицательные отступы.
  • Установите родителю font-size: 0 и затем восстановите размер шрифта для дочерних элементов.

Семантическое использование блочных и строчных элементов

  1. Когда использовать блочные элементы
    • Для структурирования больших блоков контента.
    • Когда необходимо разделить содержимое на логические секции, которые должны занимать всю ширину родительского контейнера.
    • Примеры: заголовки, абзацы, секции сайта, карточки товаров.
  2. Когда использовать строчные элементы
    • Для стилизации или выделения частей текста.
    • Когда нужно разместить элементы (например, ссылки или кнопки) в одной строке.
    • Примеры: выделение текста жирным или курсивом, создание гиперссылок, инлайновые иконки.

Современные подходы к верстке с использованием Flexbox и Grid

С развитием CSS появились более мощные инструменты для управления расположением элементов, такие как Flexbox и CSS Grid. Они позволяют более гибко управлять блочными и строчными элементами, но понимание их основ все еще критично.

1. Flexbox для блочных и строчных элементов

Flexbox отлично подходит для упрощения управления компоновкой элементов на одной оси (горизонтальной или вертикальной):

  • Можно выровнять строчные элементы по ширине или высоте контейнера.
  • Легко изменять порядок элементов, управлять равномерным распределением пространства.

Пример с Flexbox:

  
    /*CSS*/

    .container {
      display: flex;
      justify-content: space-between;
    }
    .item {
      padding: 10px;
      background-color: lightgrey;
    }

2. CSS Grid для сложных макетов

Grid упрощает создание сложных сеток и позволяет управлять как блочными, так и строчными элементами в двух измерениях (по строкам и столбцам).

Пример использования Grid:

  
    /*CSS*/

    .grid-container {
      display: grid;
      grid-template-columns: 1fr 1fr 1fr;
      gap: 10px;
    }
    .grid-item {
      padding: 20px;
      background-color: lightgreen;
    }

Заключение

Знание различий между блочными и строчными элементами поможет вам создавать семантически правильную и удобную в обслуживании разметку. Используйте их разумно, учитывая задачи и требования вашего проекта. Также не забывайте про современные методы компоновки, такие как Flexbox и Grid, которые помогают решать более сложные задачи верстки.
