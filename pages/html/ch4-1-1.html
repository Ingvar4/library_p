Основные структурные элементы: <header>, <footer>, <nav>, <main>, <article>, <section>, <aside>.

Основные структурные элементы HTML — это семантические теги, которые помогают обозначить различные разделы веб-страницы, улучшая её структуру, доступность и SEO. Вот полный обзор ключевых структурных элементов, которые широко используются для создания логичной и понятной разметки веб-страниц:

1. Тег <header>

Тег <header> используется для определения заголовочной части веб-страницы или её раздела. Обычно он включает такие элементы, как:

  • Логотип,
  • Название сайта,
  • Навигационное меню,
  • Заголовки и подзаголовки.

Пример:

  
    <header>
      <h1>Мой сайт</h1>
      <nav>
        <ul>
          <li><a href="#">Главная</a></li>
          <li><a href="#">О нас</a></li>
          <li><a href="#">Контакты</a></li>
        </ul>
      </nav>
    </header>

Особенности:

  • Тег <header> можно использовать не только в начале страницы, но и для каждого отдельного раздела (например, в статье), где требуется заголовок.
  • <header> не должен содержать других структурных тегов, таких как <footer>, но может включать <h1>–<h6>, <nav>, и другие элементы.

2. Тег <footer>

Тег <footer> предназначен для размещения информации о "подвале" страницы или её раздела. Обычно там находятся:

  • Авторские права,
  • Ссылки на политику конфиденциальности,
  • Контактные данные,
  • Ссылки на социальные сети.

Пример:

  
    <footer>
      <p>© 2024 Мой сайт. Все права защищены.</p>
      <nav>
        <ul>
          <li><a href="#">Политика конфиденциальности</a></li>
          <li><a href="#">Контакты</a></li>
        </ul>
      </nav>
    </footer>

Особенности:

  • Как и <header>, этот тег может быть использован в контексте отдельных разделов страницы, а не только в конце всего документа.
  • <footer> обычно содержит информацию о владельце сайта, ссылки на важные страницы, и другие вспомогательные элементы.

3. Тег <nav>

Тег <nav> используется для определения навигационных ссылок на веб-странице. Это может быть главное меню, ссылки на разделы или любую другую навигацию по сайту.

Пример:

  
    <nav>
      <ul>
        <li><a href="#">Главная</a></li>
        <li><a href="#">Услуги</a></li>
        <li><a href="#">Контакты</a></li>
      </ul>
    </nav>

Особенности:

  • <nav> используется только для основных навигационных элементов. Не следует помещать все ссылки страницы в этот тег.
  • Он может находиться в разных частях страницы — как в заголовке, так и в футере, или в других местах в зависимости от структуры.

4. Тег <main>

Тег <main> обозначает основное содержимое страницы, которое является уникальным для данного документа. Всё, что относится к основному контенту страницы (статьи, продукты, текст), помещается внутрь этого тега.

Пример:

  
    <main>
      <article>
        <h2>Заголовок статьи</h2>
        <p>Текст статьи...</p>
      </article>
    </main>

Особенности:

  • На странице может быть только один тег <main>.
  • Он не должен включать такие элементы, как боковые панели, футеры или навигационные элементы, если они не связаны с основным контентом.

5. Тег <article>

Тег <article> используется для обозначения независимого, самодостаточного фрагмента контента, который может существовать вне контекста страницы. Примеры — это статьи, посты в блогах, новостные заметки, комментарии.

Пример:

  
    <article>
      <h2>Заголовок статьи</h2>
      <p>Текст статьи...</p>
    </article>

Особенности:

  • Можно использовать несколько <article> тегов на одной странице.
  • Каждый <article> может включать внутри себя собственные заголовки, параграфы и другие структурные элементы.

6. Тег <section>

Тег <section> представляет собой раздел документа, который объединён общей темой или смыслом. Это может быть часть статьи, страницы или другого элемента.

Пример:

  
    <section>
      <h2>Раздел 1</h2>
      <p>Текст, относящийся к этому разделу.</p>
    </section>

Особенности:

  • Тег <section> может содержать собственные заголовки (<h1>–<h6>), а также другие теги, такие как параграфы, списки и даже другие <section> или <article>.
  • Этот тег используют для логической группировки информации. Однако если контент внутри <section> может быть независимым и использоваться отдельно, лучше применить тег <article>.

7. Тег <aside>

Тег <aside> представляет собой содержимое, которое относится к основной части страницы, но при этом является вспомогательным. Обычно используется для боковых панелей, дополнительных ссылок, рекламы, цитат или других не основных элементов.

Пример:

  
    <aside>
      <h3>Популярные статьи</h3>
      <ul>
        <li><a href="#">Статья 1</a></li>
        <li><a href="#">Статья 2</a></li>
      </ul>
    </aside>

Особенности:

  • Содержимое <aside> обычно не является ключевым для всей страницы, но поддерживает или дополняет основное содержимое.
  • Он может быть размещён как внутри тега <main>, так и за его пределами, если контент является второстепенным по отношению к основному.

Семантика и её важность

Семантические теги, такие как <header>, <footer>, <nav>, <main>, <article>, <section>, и <aside>, помогают поисковым системам и вспомогательным технологиям (таким как экранные читалки) лучше понимать структуру страницы. Они делают код более понятным для разработчиков и обеспечивают лучшую доступность.

Преимущества использования семантических тегов:

  • SEO: Поисковые системы лучше индексируют страницы, когда понимают, какие части документа имеют основное содержание, а какие — вспомогательное.
  • Доступность: Семантические теги облегчают навигацию для пользователей с ограниченными возможностями.
  • Читаемость кода: Использование структурных тегов делает HTML-документ более логичным и понятным для других разработчиков, что облегчает поддержку и редактирование кода.

Заключение

Знание и использование структурных элементов HTML крайне важно для создания современных, доступных и хорошо оптимизированных веб-страниц. Эти теги помогают улучшить как восприятие сайта пользователями, так и его эффективность в плане SEO.
