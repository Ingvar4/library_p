Доступность (Accessibility) в веб-разработке

Доступность — это подход к разработке, который делает сайты понятными и удобными для всех пользователей, включая тех, кто использует специальные устройства или программное обеспечение. Следование принципам доступности не только повышает удобство для пользователей, но и положительно влияет на SEO и общее восприятие сайта.

1. Семантические HTML-теги

Семантические теги помогают браузерам и скринридерам понимать структуру страницы.

  • Заголовки (<h1> - <h6>): Использование заголовков в порядке важности делает текст удобочитаемым и помогает пользователям быстрее ориентироваться в содержимом.
  • Основные секции страницы: <header>, <nav>, <main>, <footer> помогают понять структуру, задавая семантические области сайта.
  
    <header>Заголовок страницы</header>
    <main>Основное содержание</main>
    <footer>Контактная информация</footer>

2. aria-label, aria-labelledby, и aria-describedby

aria (Accessible Rich Internet Applications) атрибуты дают дополнительную информацию для скринридеров, улучшая взаимодействие с элементами, которые могут быть трудны для понимания только по их семантической разметке.

  • aria-label используется для добавления описания элементу, например кнопке, если текст на ней отсутствует.
    •   
    <button aria-label="Закрыть окно">✖</button>
  • aria-labelledby связывает элемент с другим элементом, который будет использоваться для озвучивания названия. Это удобно для форм.
    •   
    <label id="username">Имя пользователя</label>
    <input type="text" aria-labelledby="username">
  • aria-describedby добавляет описание элементу, часто используется для ввода данных, чтобы добавить пояснение.
    •   
    <input type="text" aria-describedby="username-description">
    <small id="username-description">Введите имя, указанное при регистрации.</small>

3. Альтернативный текст для изображений (alt)

Каждое изображение должно иметь атрибут alt, описывающий его содержание. Это помогает людям с нарушением зрения понять смысл изображения.

  
    <img src="logo.png" alt="Логотип компании">
    <img src="sunset.jpg" alt="Закат на берегу моря">

Если изображение является чисто декоративным, alt может быть пустым (alt=""), чтобы скринридеры его игнорировали.

4. Цвета и контраст

Для пользователей с нарушением зрения важно поддерживать достаточный контраст текста на фоне.

  • Контраст текста и фона: Рекомендуется соблюдать минимальное соотношение 4.5:1 для текста и 3:1 для больших заголовков. Проверить можно с помощью инструментов, таких как WebAIM Contrast Checker.
    •   
    .main-text {
      color: #000000;
      background-color: #FFFFFF;
    }
  • Избегайте зависимости от одного только цвета. Например, ошибки в форме можно помечать не только красным цветом, но и значком или текстом.

5. Клавиатурная навигация

Сайт должен быть полностью доступен через клавиатуру, без использования мыши. Это важно для пользователей с ограниченной подвижностью.

  • Порядок перехода (tabindex): Следите за логическим порядком элементов на странице. tabindex="0" устанавливает порядок по умолчанию, но избегайте присваивания других значений (кроме особых случаев), так как это может сбить естественный поток.
  • :focus стили: Элементы, на которых находится фокус, должны быть визуально заметными.
    •   
    button:focus {
      outline: 2px solid #0000FF;
    }

6. Текстовые элементы и крупный текст

Для людей с нарушениями зрения критично поддерживать возможность масштабирования текста без потери его четкости или удобочитаемости.

  • Отзывчивый текст: Используйте относительные единицы (em, rem) для размера шрифта, чтобы обеспечить масштабируемость.
  • Заголовки и текстовый контент: Текстовая иерархия должна быть четкой, что помогает всем пользователям понять смысловую структуру страницы.

7. Формы и доступность

Правильно оформленные формы делают взаимодействие удобным и понятным для всех.

  • Метки полей (<label>): Каждое поле ввода должно быть связано с меткой, чтобы скринридеры могли озвучить цель поля.
    •   
    <label for="email">Электронная почта</label>
    <input type="email" id="email" name="email">
  • Поле обязательного заполнения: Если поле обязательно для заполнения, добавьте aria-required="true".
    •   
    <input type="text" aria-required="true">
  • Сообщения об ошибках: Добавляйте aria-live="assertive" для сообщений об ошибках, чтобы они автоматически озвучивались скринридерами.

8. Фокус и управление всплывающими окнами

Для элементов, таких как модальные окна, требуется особая работа с фокусом.

  • Фокус на модальных окнах: После открытия модального окна перенаправляйте фокус на первое интерактивное поле или кнопку внутри него.
  • Возврат фокуса: При закрытии окна возвращайте фокус обратно на элемент, который изначально его открыл.
    •   
    const modalTrigger = document.querySelector('#open-modal');
    const modal = document.querySelector('#modal');

    modalTrigger.addEventListener('click', () => {
        modal.style.display = 'block';
        modal.querySelector('input').focus(); // Фокус на первое поле
    });

    modal.querySelector('.close').addEventListener('click', () => {
        modal.style.display = 'none';
        modalTrigger.focus(); // Возврат фокуса
    });

9. ARIA-ролы и атрибуты

ARIA-ролы помогают улучшить доступность нестандартных элементов интерфейса (навигационных меню, кнопок, всплывающих окон), делая их понятными для скринридеров.

  • Роль кнопки для нестандартного элемента: Если вы создаёте элемент кнопки на основе <div>, добавьте role="button".
    •   
    <div role="button" tabindex="0" aria-pressed="false">Переключить</div>
  • Роль навигации: Помогает пользователям быстро переходить к основным разделам.
    •   
    <nav role="navigation">
      <ul>
          <li><a href="#home">Главная</a></li>
          <li><a href="#services">Услуги</a></li>
      </ul>
    </nav>

Заключение

Следование этим принципам и стандартам помогает создать веб-интерфейс, доступный всем пользователям.
