Статические метаданные

Представляют собой данные title и description в layout.tsx

Но проблема в том, что метаданные из корневого layout отображаются и на других страницах.

Пример изменения метаданных на странице about > page.tsx

  
    import Link from "next/link";

    export const metadata = {
      title: 'Страница о компании',
      description: 'Информация о нашей компании...'
    }

    export default function AboutPage() {
      return (
        <>
          <h1>Страница о компании</h1>
          <Link href="/">На главную</Link>
        </>
      );
    }

То есть мы экспортируем metadata и заполняем её для нужной нам страницы

Динамические метаданные

Для того, чтобы заполинть метадату для страниц, которые генерируются динамически, к примеру [productId], используется функция:

src/app/products/[productId]/page.tsx

  
    import type { Metadata } from "next";

    type Props = {
      params: Promise<{ productId: string }>;
    };

    export const generateMetadata = async ({
      params,
    }: Props): Promise<Metadata> => {
      const id = (await params).productId;

      return {
        title: `Продукт ${id}`,
        description: `Продукт ${id} - хит продаж`,
      };
    };

    export default async function ProductDetails({ params }: Props) {
      const productId = (await params).productId;
      return <h1>Описание продукта {productId}</h1>;
    }

Динамические метаданные, извлекаемые из API

Пример с src/app/posts/[postId]/page.tsx

  
    import { Metadata } from "next";

    interface Post {
      id: number;
      title: string;
      body: string;
    }

    type Props = {
      params: Promise<{ postId: string }>;
    };

    async function getPost(id: string): Promise<Post> {
      try {
        const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
        if (!res.ok) throw new Error(`Ошибка ответа сервера: ${res.status}`);
        return await res.json();
      } catch (error) {
        console.error(`Не удалось получить данные:`, error);
        throw error;
      }
    }

    export const generateMetadata = async ({
      params,
    }: Props): Promise<Metadata> => {
      const { postId } = await params;
      const post = await getPost(postId);

      return {
        title: `${post.title} | My Blog`,
        description: post.body.slice(0, 160),
      };
    };

    export default async function PostPage({ params }: Props) {
      const { postId } = await params;

      const post = await getPost(postId);

      return (
        <div>
          <h1 className="text-2xl">{post.title}</h1>
          <p>{post.body}</p>
        </div>
      );
    }

ВАЖНО: на странице нужно использовать только один тип метаданных. Либо статический, либо динамический. Объединять нельзя.

Так же: метаданные не работают в клиентских компонентах c use client

Выход такой, что рядом с клиентским компонентом создаётся серверный page.tsx. В page.tsx импортируется клиентский компонет и заполняются метаданные

Продвинутая работа с заголовками страниц

корневой layout

  
    export const metadata: Metadata = {
      title: {
        default: "Мое приложение на Next.js",
        template: "%s | Мое приложение на Next.js",
      },
      description: "Я изучаю Next.js последней версии",
    };

"%s означает, что на дочерней странице будет отображен сначала заголовок этой страницы (если он задан), а затем | Мое приложение на Next.js"

default соотв. отображается если заголовок дочерней страницы не задан

Свойство absolute

Если нужно игнорировать метазаголовок корневого layout, то на дочерней странице используется метод absolute. И тогда будет отображаться заголовок исключительно этой странцы, без заголовка корневого layout

  
    export const metadata = {
      title: {
        absolute: "Наши контакты",
      },
      description: "Нас можно найти...",
    };

    export default function Contacts() {
      return (
        <div>
          <p>Contacts</p>
          <p>+1234567890</p>
        </div>
      );
    }
